# Updated Portfolio

## Project Overview

This is the new personal portfolio for **Mohit Manna**, a Computer Engineering student at Purdue University building across software engineering, systems, applied intelligence, infrastructure, data, and product-facing engineering.

This portfolio should not feel like a standard student portfolio.

It should feel like a **minimal technical artifact**: quiet, premium, interactive, and memorable.

The goal is to translate Mohit’s resume, technical experience, and project history into a clean visual experience. The website should not dump every resume bullet onto the page. The resume is the data source. The website is the experience.

Core feeling:

> Static at first glance. Alive when touched. Deep when explored.

The site should feel:

- minimal
- high-aura
- editorial
- premium
- technical
- calm
- nonchalant
- recruiter-friendly
- smooth
- low-latency
- intentional

The website should make recruiters and engineers think:

> This person has technical depth, strong taste, and knows how to present engineering work like a product.

Avoid making this site feel like:

- a generic dark neon developer portfolio
- a basic resume website
- a startup SaaS landing page
- a template portfolio with random project cards
- an overanimated Three.js experiment
- a cluttered skill-icon dump
- a corporate LinkedIn clone
- a “welcome to my portfolio” student site

The portfolio should feel sharp, controlled, and personal.

---

# Creative North Star

## Main Concept

The main visual identity of this portfolio is a **scroll-driven systems chip teardown**.

The homepage hero should feature a futuristic black graphite and gold silicon chip. The chip visually represents Mohit’s engineering identity and technical stack.

The chip starts assembled.

As the user scrolls, the video scrubs forward and reveals the chip deconstructing into layered components.

This is the central metaphor:

```text
Mohit = the system
chip layers = the technical stack
scroll = revealing depth
```
